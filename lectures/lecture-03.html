Search Intent - фундамент SEO

Search Intent (User Intent) - намір, з яким користувач вводить запит у пошукову систему.

Google's Mission: "Організувати світову інформацію та зробити її загальнодоступною та корисною"
RankBrain (2015) + BERT (2019) + MUM (2021): Google розуміє НЕ слова, а НАМІР користувача
Статистика:
  • 15% щоденних запитів Google ніколи раніше не бачив
  • 70% користувачів клікають на результати, що відповідають їхньому intent
  • Неправильний intent → bounce rate 80%+ → втрата позицій

4 типи Search Intent

  • Informational Intent (Інформаційний)
  • Navigational Intent (Навігаційний)
  • Commercial Investigation (Комерційне дослідження)
  • Transactional Intent (Транзакційний)

SERP (Search Engine Results Page) характеристики

SERP характеристики - це особливості або елементи сторінки результатів пошуку в пошукових системах.

Коли користувач вводить запит у пошукову систему, наприклад у Google Search, система показує не лише звичайні посилання на сайти, а й різні додаткові блоки. Саме ці блоки і називають SERP features (SERP-характеристики).

Найпоширеніші елементи SERP характеристик:
  • Featured Snippet - блок із короткою відповіддю на запит (часто на самому верху сторінки)
  • Knowledge Panel - інформаційна панель про людину, компанію чи об’єкт
  • Local Pack - карта з місцевими компаніями
  • Image Pack - блок із зображеннями
  • Video results - результати відео (часто з YouTube)
  • People Also Ask - блок «Люди також запитують»
  • Top Stories - блок новин
  • Shopping results - товари з цінами
  • Site links - додаткові посилання всередині одного сайту

Навіщо це важливо? У сфері SEO (Search Engine Optimization) оптимізація під SERP features допомагає:
  • отримати більшу видимість у пошуку

Informational Intent (Інформаційний)

Намір - дізнатися інформацію, навчитися, зрозуміти

Приклади запитів:
  • як працює React hooks
  • що таке SEO
  • історія JavaScript
  • рецепт борщу
Типові модифікатори:
  • як, що, чому, коли, де, хто
  • гайд, інструкція, tutorial, приклад
  • порівняння, різниця між
Google показує SERP характеристики:
  • Featured Snippets (позиція 0)
  • People Also Ask
  • Статті (blogs, guides)
  • Відео з YouTube
  • Knowledge Graph (для фактів)
  • Ads: мінімум (low commercial value)

Намір - знайти конкретний сайт або сторінку

Google показує SERP характеристики:
  • Офіційний сайт бренду (позиція 1)
  • Sitelinks (підсторінки)
  • Knowledge Panel (праворуч)

Commercial Investigation (Комерційний)

Намір - вибір перед покупкою, порівняння варіантів

Приклади запитів:

  • MacBook Pro vs Dell XPS

  • порівняння React vs Vue
  • який фреймворк вибрати
Google показує SERP характеристики:
  • Огляди (reviews, listicles)
  • Порівняльні статті
  • Відео-огляди (YouTube)
  • E-commerce (позиції 7-10)
  • Ads - shopping ads, comparison ads

Transactional Intent (Транзакційний)

Намір - здійснити дію (купити, завантажити, замовити)

Приклади запитів:
  • купити iPhone 15 Pro Чернівці
  • замовити піцу додому
  • скачати VS Code

  • купити ноутбук онлайн
Типові модифікатори:
  • купити, замовити, придбати, онлайн
  • інтернет-магазин, ціна, вартість, знижка
  • доставка, безкоштовно
Google показує SERP характеристики:
  • E-commerce сайти (TOP-3)
  • Shopping Ads (зверху) - Ads dominated (60-70% SERP)
  • Google Shopping (праворуч)
  • Local Pack (для локальних запитів)
  • Price comparison sites

Семантичне ядро - визначення та структура

Семантичне ядро (Semantic Core) - упорядкований список всіх ключових слів, за якими планується просування сайту в пошукових системах.

HIGH-FREQUENCY (ВЧ) - 10,000+ запитів/міс
  • Високий обсяг трафіку
  • Висока конкуренція (KD 60-90)
  • Низька конверсія (1-2%)
  • Загальні запити
  • Приклад: "ноутбук", "смартфон"
MID-FREQUENCY (СЧ) - 1,000-10,000 запитів/міс
  • Помірний обсяг
  • Середня конкуренція (KD 30-60)
  • Середня конкуренція (KD 30-60)
  • Хороша конверсія (2-4%) │
  • Специфічніші запити
  • Приклад: "ігровий ноутбук", "ноутбук для студента"
LOW-FREQUENCY / LONG-TAIL (НЧ) - 10-1,000 запитів/міс
  • Малий обсяг
  • Низька конкуренція (KD 5-30)
  • ВИСОКА конверсія (5-15%)
  • Дуже специфічні
  • Приклад: "купити ноутбук ASUS ROG, Strix G15 київ доставка"

Семантичне ядро - визначення та структура

Правило Парето (80/20) в SEO:

  • 20% ВЧ keywords → 80% impressions, 20% conversions
  • 80% НЧ keywords → 20% impressions, 80% conversions
Фокус на Long-Tail для конверсій!

Silo-структура сайту

Silo Architecture - організація контенту в ізольовані тематичні блоки для максимальної релевантності.

Традиційна структура (проблемна) - на прикладі головної сторінки сайту:
  • Товар 1 ↔ Товар 2 ↔ Товар 3
  • Стаття 1 ↔ Стаття 2
  • Категорія A ↔ Категорія B
  • (хаотичні посилання між різними темами)
Проблеми:
  • Google не розуміє, яка тема домінує
  • PageRank розпорошується
  • Weak topical authority
  • Keyword cannibalization

Silo-структура сайту

  • Головна
    • SILO 1: Ігрові ноутбуки
      • Hub: /gaming-laptops/
      • Category: /gaming-laptops/budget/

Silo-структура сайту

Правила:
  • Перелінковка ТІЛЬКИ всередині силосу
  • Між силосами ТІЛЬКИ через головну
  • Кожен силос = кластер keywords
  • Чітка тематична релевантність

Типи Silo-структур

Physical Silo (Фізична) - через URL структуру (URL hierarchy reflects silo structure)
  • /gaming-laptops/ ← SILO 1
    • /gaming-laptops/ (hub)
    • /gaming-laptops/budget/
    • /gaming-laptops/premium/
    • /gaming-laptops/asus-rog/
    • /gaming-laptops/guides/buying/
  • /business-laptops/ ← SILO 2
    • /business-laptops/ (hub)
    • /business-laptops/ultrabooks/
    • /business-laptops/dell-xps/
Переваги:
  • Чіткість для користувачів
  • Google легко розпізнає структуру
  • Breadcrumbs природні
  • URL = site hierarchy

Типи Silo-структур

Virtual Silo (Віртуальна) - через internal linking
  • https://techstore.ua/
    • /gaming-laptops (hub, links to all gaming pages)
    • /asus-rog (links ONLY to gaming hub + related gaming)
    • /msi-raider (links ONLY to gaming hub + related gaming)
    • /gaming-buying-guide (links ONLY to gaming pages)
    • /business-laptops (hub, separate silo)
    • /dell-xps (links ONLY to business hub)
    • NO links between /asus-rog ↔ /dell-xps

Типи Silo-структур

Hybrid Silo (Гібридна) - комбінація physical для основи + virtual для flexibility
  • https://techstore.ua/
    • /gaming-laptops/ (physical silo)
      • /gaming-laptops/asus-rog/
      • /gaming-laptops/msi-raider/
    • /guides/ (shared content pool)
      • /guides/gaming-laptop-buying-guide/
        • Virtual links: посилається на /gaming-laptops/ + products
      • /guides/business-laptop-guide/
        • Virtual links: посилається на /business-laptops/
    • /blog/ (flexible content)
      • /blog/best-laptops-2025/
        • Може посилатись на різні silos contextually

Рекомендація для більшості проєктів - hybrid підхід дає найкращий баланс структури та гнучкості.

Internal Linking Strategy для Silo

Правила перелінковки:
  1. Hub → All Supporting Pages (Downstream Links) Hub має посилання на всі сторінки в силосі
  2. Supporting → Hub (Upstream Links) Кожна сторінка посилається назад на hub
  3. Supporting ↔ Related Supporting (Lateral Links) Між related сторінками того ж силосу
  4. NO Cross-Silo Links Між силосами ТІЛЬКИ через головну або footer
  5. Anchor Text: природний, varied, keyword-rich
